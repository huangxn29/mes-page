# 贡献指南

欢迎参与 MES-Page 项目的开发！请遵循以下规范。

## 快速开始

1. Fork 本仓库
2. 基于 `develop` 创建你的功能分支：`git checkout -b feature/mes-page-<your-feature> develop`
3. 提交代码（遵循约定式提交规范）
4. 创建 Pull Request 到 `develop` 分支

## 开发规范

详细的 Git 和 GitHub 操作规范请参阅 [GITHUB_OPERATIONS.md](GITHUB_OPERATIONS.md)，主要包括：

- **分支策略**：Git Flow 模型
- **提交信息**：约定式提交规范（Conventional Commits）
- **PR 流程**：创建 → Review → 合并
- **Code Review**：1 人 Approve 后方可合并

## 需要帮助？

- 查阅 [GITHUB_OPERATIONS.md](GITHUB_OPERATIONS.md) 获得完整的操作指南
- 在项目中创建 Issue 报告问题或提出建议
